# 实用工具聚合站 — 本地与云端部署测试指南

## 前置要求

| 环境 | 版本要求 |
|------|---------|
| Python | 3.10+ |
| pip | 最新版 |
| 浏览器 | Chrome / Edge / Safari |

---

## 一、本地部署（Windows）

### 1.1 进入项目目录

```powershell
cd "D:\vibecoding\聚合站点\utility-station-main"
```

### 1.2 安装依赖

```powershell
pip install -r requirements.txt
```

如果之前安装过旧版本，建议先清理缓存：

```powershell
pip install -r requirements.txt --force-reinstall --no-cache-dir
```

### 1.3 创建数据库目录（首次必需）

```powershell
mkdir instance
```

> SQLite 数据库文件默认存放在 `instance/utility.db`，此目录必须提前创建，否则启动会报 `unable to open database file`。

### 1.4 （可选）设置环境变量

在启动前可以通过环境变量预设管理员账号和密码。如果不设置，系统**首次启动时会自动生成一个随机密码并打印到控制台**，请留意保存。

**PowerShell 设置方式：**

```powershell
$env:ADMIN_EMAIL="admin@utility.com"
$env:ADMIN_PASSWORD="YourStrongPassword123"
$env:DEBUG="True"
```

**CMD 设置方式：**

```cmd
set ADMIN_EMAIL=admin@utility.com
set ADMIN_PASSWORD=YourStrongPassword123
set DEBUG=True
```

> **⚠️ 重要提示（Windows 必看）：**
> - **环境变量仅对当前终端会话有效**。关闭 PowerShell / CMD 窗口后环境变量会丢失，下次打开新窗口需要重新设置。
> - **必须在同一终端窗口中设置变量并启动应用**。例如：先执行上面的 `$env:...` 命令，然后紧接着在同一窗口执行 `python app.py`。
> - 如果通过 VS Code 的「运行」按钮或双击 `.py` 文件启动，终端中设置的环境变量**不会传递**到 Python 进程。请始终通过终端命令 `python app.py` 启动。
> - **管理员密码只会创建时生效一次**。如果之前已经启动过（数据库中已有管理员），之后即使修改 `ADMIN_PASSWORD` 环境变量，系统**会自动检测到并更新密码**。启动时控制台会打印 `[诊断] 当前管理员密码来源: 环境变量` 来确认。
> - **DEBUG 模式影响验证码显示**。`DEBUG=True` 时发送验证码会直接在前端提示框显示明文验证码（方便本地测试）；`DEBUG=False` 时验证码不会显示。

### 1.5 启动服务

```powershell
python app.py
```

默认端口为 `5000`。控制台会显示类似：

```
 * Running on http://0.0.0.0:5000
```

### 1.6 浏览器访问

打开浏览器，访问以下地址：

- **本机访问**：`http://localhost:5000` 或 `http://127.0.0.1:5000`
- **局域网访问**：`http://<你的局域网IP>:5000`（同一 WiFi 下的手机/平板可访问）

### 1.7 停止服务

在 PowerShell / CMD 中按 `Ctrl + C` 即可停止。

## 二、CloudStudio 云端部署（Linux）

CloudStudio 是腾讯云提供的云端 IDE，已内置 Python 环境，可直接运行本项目。

### 2.1 打开 CloudStudio 工作空间

1. 进入 [CloudStudio 控制台](https://cloudstudio.net/)
2. 创建或打开一个基于 **Python** 或 **Ubuntu** 模板的空工作空间
3. 将项目代码上传至工作空间根目录（可通过 Git 克隆或直接拖拽上传）

### 2.2 一键部署

项目根目录已内置 `deploy.sh` 一键部署脚本，在终端中执行：

```bash
sh deploy.sh
```

脚本会自动完成以下操作：
1. 安装 `requirements.txt` 依赖
2. 创建 `instance` 数据库目录
3. 初始化 SQLite 数据表
4. 后台启动 Flask 服务（端口 `3000`）

### 2.3 查看部署输出

脚本执行成功后，终端会显示：

```
========================================
  ✅ 部署完成！
========================================

  访问地址: http://localhost:3000

  Cloud Studio 会自动将端口映射为可访问链接
  点击底部「端口」面板中的链接即可打开

  停止服务: bash stop.sh
========================================
```

### 2.4 使用端口映射访问

CloudStudio 会自动检测并映射 `3000` 端口：

1. 点击编辑器底部状态栏的 **「端口」** 面板
2. 找到 `3000` 端口对应的映射链接（如 `https://xxx-3000.app.cloudstudio.net`）
3. 点击即可在浏览器中打开应用

### 2.5 查看运行日志

```bash
tail -f /tmp/utility.log
```

按 `Ctrl + C` 退出日志追踪。

### 2.6 停止服务

```bash
bash stop.sh
```

或直接查找进程并终止：

```bash
pkill -f "python app.py"
```

### 2.7 重新部署（修改源码后）

修改源码后，需要按以下步骤重新部署到 CloudStudio：

#### 场景 A：本地修改后推送到 GitHub，云端拉取更新

适用于：在本地 VS Code 修改代码，推送到 GitHub，然后在 CloudStudio 更新。

```bash
# 1. 进入项目目录
cd ~/utility-station-main

# 2. 拉取最新代码（确保在 main 分支）
git checkout main
git pull origin main

# 3. 停止旧服务
bash stop.sh

# 4. 重新部署
bash deploy.sh
```

#### 场景 B：直接在 CloudStudio 修改代码

适用于：直接在 CloudStudio 编辑器中修改代码。

```bash
# 1. 进入项目目录
cd ~/utility-station-main

# 2. 停止旧服务
bash stop.sh

# 3. 重新部署
bash deploy.sh
```

#### 场景 C：首次部署到新的 CloudStudio 工作空间

适用于：全新环境从零开始部署。

```bash
# 1. 克隆项目（替换为你的仓库地址）
git clone https://github.com/June-Yan/SC30-4ever-utility-station.git
cd SC30-4ever-utility-station

# 2. 一键部署
bash deploy.sh
```

> **部署后检查清单：**
> - `bash stop.sh && bash deploy.sh` 后无报错
> - `tail -f /tmp/utility.log` 中无异常堆栈
> - 底部「端口」面板中 `3000` 端口已映射
> - 浏览器打开映射链接能正常访问首页
> - 管理员密码已按预期设置（检查日志中的 `[诊断] 管理员密码来源`）

---

### 2.8 覆盖原有配置（.env 与环境变量）

CloudStudio 云端部署时，`.env` 文件**默认不会被 Git 跟踪**（已在 `.gitignore` 中排除）。如果需要设置管理员密码等配置，请在 CloudStudio 终端中手动创建或编辑 `.env`：

```bash
# 进入项目目录
cd ~/utility-station-main

# 创建/编辑 .env
nano .env
```

`.env` 示例配置：

```env
SECRET_KEY=你的密钥（首次启动自动生成，无需修改）
ADMIN_EMAIL=admin@utility.com
ADMIN_PASSWORD=YourStrongPassword123
```

> **⚠️ 重要提示：**
> - `.env` 文件包含敏感信息，**切勿提交到 Git**
> - 修改 `.env` 后必须**重启服务**才能生效：`bash stop.sh && bash deploy.sh`

## 三、测试清单

部署成功后，请按以下清单逐一验证本次二次开发的四个核心功能。

### 3.1 首页搜索

1. 打开首页
2. 在搜索框输入关键词，例如：`计算器`、`天气`、`备忘`
3. 观察下方是否出现实时匹配的搜索结果
4. 点击搜索结果，确认能正确跳转页面或打开小组件弹窗

### 3.2 管理员面板

1. **首次启动时**，注意控制台输出的管理员账号和密码，格式如下：
   ```
   邮箱: admin@utility.com
   密码: xxxxxxxx
   ```
2. 使用管理员邮箱和密码登录
3. 观察导航栏是否出现 **⚙️ 管理** 入口（仅管理员可见）
4. 点击进入管理面板，验证以下模块：
   - **数据概览**：显示待审核作品数、待回复反馈数、注册用户、点击数据等
   - **作品审核**：查看待审核的创意工坊作品，可进行通过 / 拒绝 / 删除操作
   - **反馈处理**：查看用户反馈列表，对待回复的反馈进行文字回复

### 3.3 星级评分

1. 进入 **🏆 排行** 页面
2. 点击 **⭐ 星级排行** 标签页，查看是否有星级数据（首次为空是正常的）
3. 进入 **🧩 工具箱 → 预设工具**，任意点击一个小组件（如计算器）
4. 在弹窗底部找到 **⭐⭐⭐⭐⭐** 评分区域
5. 点击星星进行评分（1~5 星）
6. 再次进入星级排行，确认该小组件出现且平均分正确
7. 重新打开该小组件，确认已保存的评分状态被正确回显

### 3.4 游客模式强制登录

1. 退出当前账号，进入 **游客模式**
2. 在首页点击 **🧩 工具箱**
3. 点击 **✨ 我的自定义** 标签页
4. 确认出现 **🔒 需要登录才能使用自定义小组件** 的提示
5. 点击登录按钮后，确认跳转到登录页
6. 此行为应与备忘录的强制登录逻辑一致

### 3.5 验证码登录

1. 进入登录页，点击 **验证码登录** 标签
2. 输入任意邮箱地址（无需真实邮箱）
3. 点击 **获取验证码** 按钮
4. 确认页面下方出现 **验证码：XXXXXX** 的提示（6 位数字）
5. 将验证码输入到验证码输入框，点击登录
6. 确认登录成功，进入首页

> 验证码不需要真实邮箱，系统生成后直接显示在页面上，简单高效。

### 3.6 URL 类型小组件双按钮交互

1. 进入 **🧩 工具箱 → ✨ 我的自定义**
2. 添加一个 **URL 类型**小组件（填写名称、图标、目标链接）
3. 保存后，在"我的自定义"列表中找到该卡片：
   - 确认卡片**不会自动跳转**（`cursor: default`）
   - 确认卡片底部出现两个按钮：**🔗 打开链接** 和 **🗑️ 删除**
   - 确认卡片显示星级评分区域（⭐）
4. 点击 **🔗 打开链接**，确认在新标签页打开目标 URL
5. 点击 **⭐ 星星** 进行评分，刷新页面后确认评分已保存
6. 进入 **🧑‍🎨 创意工坊**，找到任意已添加的 URL 类型社区作品：
   - 确认卡片同样显示 **🔗 打开链接** 按钮和星级评分
   - 确认不会点击卡片就跳转
7. 点击 URL 类型社区作品卡片，在弹窗中确认：
   - 弹窗内显示链接预览和 **打开链接** 按钮
   - 弹窗底部有星级评分区域

## 四、常见问题排查

| 问题现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `unable to open database file` | `instance` 目录不存在 | 手动创建 `mkdir instance` |
| 管理员密码忘记 | 首次启动后未保存随机密码 | 删除 `instance/utility.db` 后重新启动，系统会重新生成管理员账号 |
| **设置了 `ADMIN_PASSWORD` 但仍显示密码错误** | 之前已启动过，数据库中存的是旧密码；或环境变量未传递到 Python 进程 | 在同一终端中设置环境变量后执行 `python app.py`。启动时会打印 `[诊断] 当前管理员密码来源: 环境变量`。若仍不对，删除 `instance/utility.db` 重新启动 |
| 前端样式未更新 | 浏览器缓存了旧版 CSS/JS | 按 `Ctrl + F5` 强制刷新，或修改 `index.html` 中的 `?v=N` 版本号 |
| CloudStudio 端口映射不生效 | 服务未在 `3000` 端口运行 | 检查 `deploy.sh` 中 `PORT=3000` 是否正确，或手动指定端口启动 |

## 五、环境变量速查表

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `ADMIN_EMAIL` | `admin@utility.com` | 预设管理员邮箱 |
| `ADMIN_PASSWORD` | 随机生成 | 预设管理员密码（未设置则启动时自动生成并打印） |
| `DEBUG` | `False` | 调试模式（`True` 时发送验证码会直接在前端提示框显示） |
| `PORT` | `5000` | 服务监听端口 |
| `SECRET_KEY` | 自动生成 | JWT 签名密钥（首次启动自动写入 `.env`） |
| `JWT_EXPIRY_DAYS` | `7` | JWT Token 有效期（天） |
| `DATABASE_URL` | SQLite | 数据库连接字符串（默认使用本地 SQLite） |
